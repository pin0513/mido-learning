# Handoff Protocol

## 適用範圍

所有交付給 Web-Dev-Team 的設計專案都必須遵守此協議。

---

## 交接時機

**觸發條件**（三者必須全部滿足）：
1. 所有程式碼通過 QA 驗收
2. 客戶最終簽核完成
3. 所有審核報告已產出

**禁止提前交接**：即使客戶催促，未完成品質檢查不得交接。

---

## 交接檢查清單

### 程式碼交付物
- [ ] HTML/React 程式碼（可執行）
- [ ] CSS 樣式檔案
- [ ] 所有圖片資源（WebP 格式）
- [ ] SVG 圖標檔案
- [ ] package.json（若使用 npm）

### 設計文件
- [ ] 最終版 Figma 連結（桌面版 + 行動版）
- [ ] 設計規範文件（顏色、字級、間距）
- [ ] RWD 斷點說明

### 審核報告
- [ ] SEO 技術審核報告
- [ ] Web UI 品管審核報告
- [ ] RWD 驗收報告
- [ ] Core Web Vitals 測試結果

### 客戶資料
- [ ] 客戶 feedback 紀錄（2 輪）
- [ ] 最終簽核證明

---

## 交接會議

### 會議前準備（交接前 1 天）
1. 將所有檔案上傳至共享目錄
2. 產生交接文件（使用 `/design-handoff` skill）
3. 通知 Web-Dev-Team 會議時間

### 會議議程（30-45 分鐘）
1. **設計意圖說明**（10 分鐘）：設計總監或 UI 設計師說明設計理念
2. **技術細節說明**（15 分鐘）：UI 設計師說明程式碼結構、特殊實作
3. **問題答疑**（10 分鐘）：Web-Dev-Team 提問
4. **後續支援確認**（5 分鐘）：確認支援窗口與聯絡方式

### 會議產出
- 交接會議紀錄（Markdown 格式）
- 待釐清問題清單（若有）

---

## 交接文件格式

```markdown
# 交接文件：{專案名稱}

**交接日期**：{YYYY-MM-DD}
**專案類型**：企業官網 / Landing Page
**負責設計師**：{姓名}
**接收團隊**：Web-Dev-Team

## 設計重點
- {設計意圖}

## 技術實作重點
- {特殊技術說明}

## 已知限制
- {若有限制或待改進項目}

## 支援窗口
- 設計支援：{姓名}
- 技術支援：{姓名}
```

---

## 違反判定

- 未完成 QA 驗收即交接 → 違反
- 缺少任何必要交付物 → 違反
- 未舉辦交接會議 → 違反
- 未產生交接文件 → 違反
