# Design Standards

## 適用範圍

所有 Web Design Team 的設計專案都必須遵守此標準。

---

## Layout & Spacing

### 最大內容寬度
- **桌面版**：1200px
- **違反範例**：內容區塊寬度超過 1200px，導致超寬螢幕上內容過度分散

### 容器 Padding
- **行動版**：左右各 20px
- **桌面版**：左右各 40px
- **違反範例**：文字緊貼螢幕邊緣

### 間距系統（8px 基準）
- **元素內間距**：8px, 16px
- **元素間間距**：24px, 32px
- **區塊間間距**：48px, 64px, 80px
- **違反範例**：使用 15px, 23px, 37px 等不規則間距

---

## Typography

### 字級大小
- **H1**：48-64px（桌面）、32-36px（行動）
- **H2**：32-42px（桌面）、28-32px（行動）
- **H3**：20-24px
- **Body**：16-18px
- **Small**：12-14px
- **違反範例**：行動版內文小於 16px

### 行距
- **最小行距**：1.5
- **建議行距**：1.6-1.8
- **違反範例**：行距 1.2，導致文字過於擁擠

### 行長
- **建議行長**：60-80 字元
- **違反範例**：桌面版行長超過 100 字元，難以閱讀

---

## Color & Contrast

### 對比度標準（WCAG AA）
- **一般文字**：4.5:1
- **大型文字（≥ 18px）**：3:1
- **UI 元件**：3:1
- **違反範例**：淺灰色文字（#999）在白色背景上（對比度 2.8:1）

### 色彩使用
- **主要顏色**：不超過 3-4 種
- **品牌色**：必須正確使用品牌指南的色碼
- **違反範例**：使用 10 種以上不同顏色，視覺混亂

---

## Responsive Design

### 斷點
```css
/* Mobile: < 768px (預設) */
@media (min-width: 768px) { /* Tablet */ }
@media (min-width: 1024px) { /* Desktop */ }
```

### 觸控目標
- **最小尺寸**：44x44px
- **違反範例**：行動版按鈕高度 32px，難以點擊

### 行動版佈局
- **原則**：Mobile-first，單欄堆疊
- **違反範例**：行動版只是桌面版縮小，沒有重新佈局

---

## Components

### 圖標
- **格式**：SVG
- **禁止**：使用 emoji（🏠）
- **違反範例**：使用 emoji 當圖標，在不同裝置上顯示不一致

### 按鈕
- **必須有**：hover 狀態
- **違反範例**：按鈕無 hover 狀態，使用者不知道可點擊

### Focus Rings
- **必須有**：明顯的 focus outline
- **禁止**：`outline: none`
- **違反範例**：使用 Tab 鍵操作時，無法判斷當前 focus 在哪裡

---

## 圖片

### 格式
- **優先使用**：WebP
- **Fallback**：PNG/JPG
- **違反範例**：使用未壓縮的 PNG，檔案大小 > 2MB

### 優化
- **檔案大小**：< 200KB
- **載入方式**：lazy loading（非首屏圖片）
- **違反範例**：首頁載入 10 張未壓縮的圖片，LCP > 5s

---

## 違反判定

- 未遵守間距系統（使用不規則間距）→ 違反
- 行動版文字小於 16px → 違反
- 對比度不足（< 4.5:1）→ 違反
- 使用 emoji 當圖標 → 違反
- 按鈕無 hover 狀態 → 違反
- 使用 `outline: none` 移除 focus rings → 違反

---

## 例外

- 特殊設計需求必須經過設計總監批准
- 客戶品牌指南與此標準衝突時，優先遵循品牌指南（但必須記錄）
