# RWD Validation Skill

## 目的

提供系統化的 RWD 驗收檢查清單，確保網站在桌面版、平板版、行動版三種裝置上都能正常顯示與操作。

---

## 測試裝置清單

### 桌面版（Desktop）
- [ ] 1920x1080（Full HD）
- [ ] 1440x900（MacBook Air）
- [ ] 1366x768（常見筆電）

### 平板版（Tablet）
- [ ] 768x1024（iPad 直立）
- [ ] 1024x768（iPad 橫向）

### 行動版（Mobile）
- [ ] 375x667（iPhone SE, iPhone 8）
- [ ] 414x896（iPhone 11, iPhone XR）
- [ ] 360x640（Android 常見尺寸）

---

## 檢查清單

### 佈局（Layout）
- [ ] 內容不會超出螢幕寬度（無橫向滾動）
- [ ] 元素不會重疊
- [ ] 間距合理（不過度擁擠或過度空曠）
- [ ] 區塊堆疊順序正確（行動版）
- [ ] 最大內容寬度正確（桌面版 1200px）

### 文字（Typography）
- [ ] 文字大小適當（行動版最小 16px）
- [ ] 行長合理（60-80 字元）
- [ ] 標題與內文對比明確
- [ ] 文字不會被截斷
- [ ] 行距舒適（1.5-1.8）

### 圖片（Images）
- [ ] 圖片自動縮放（不變形）
- [ ] 圖片載入正常
- [ ] 圖片不會超出容器
- [ ] Lazy loading 正常運作
- [ ] 圖片有 width 和 height 屬性

### 導航（Navigation）
- [ ] 桌面版：主導航顯示完整
- [ ] 行動版：漢堡選單可正常開啟/關閉
- [ ] 平板版：依設計決定（漢堡選單或完整導航）
- [ ] Logo 可點擊回首頁
- [ ] 導航項目不會被截斷

### 按鈕與連結（Buttons & Links）
- [ ] 按鈕大小適當（行動版最小 44x44px）
- [ ] 按鈕可正常點擊
- [ ] Hover 狀態正常（桌面版）
- [ ] 連結可正常點擊
- [ ] 觸控目標之間有足夠間距

### 表單（Forms）
- [ ] 輸入框大小適當
- [ ] 鍵盤彈出後不會遮蔽輸入框（行動版）
- [ ] 下拉選單可正常操作
- [ ] 提交按鈕可正常點擊
- [ ] 錯誤訊息正常顯示

### 其他互動元素
- [ ] Modal / Popup 可正常開啟/關閉
- [ ] 手風琴（Accordion）可正常展開/收合
- [ ] 輪播（Carousel）可正常滑動
- [ ] 錨點連結（Anchor Links）可正常跳轉
- [ ] 下拉選單不會超出螢幕

## 快速測試流程

### 1. 桌面版測試（10 分鐘）
```
1. 開啟 Chrome，設定視窗為 1920x1080
2. 逐頁檢查佈局
3. 點擊所有連結與按鈕
4. 填寫表單測試
5. 截圖記錄
```

### 2. 平板版測試（10 分鐘）
```
1. 使用 Chrome DevTools，設定 768x1024（直立）
2. 檢查導航變化（漢堡選單）
3. 測試觸控操作
4. 切換橫向模式（1024x768）
5. 截圖記錄
```

### 3. 行動版測試（15 分鐘）
```
1. 測試 iPhone SE（375x667）
2. 測試 iPhone 11（414x896）
3. 測試 Android（360x640）
4. 檢查單欄堆疊佈局
5. 測試漢堡選單
6. 測試表單與鍵盤互動
7. 截圖記錄
```

## 常見問題參考

### 橫向滾動
- 原因：固定寬度元素
- 檢查：`width: 1200px` 改為 `max-width: 1200px`

### 觸控目標過小
- 原因：按鈕高度 < 44px
- 檢查：`min-height: 44px`

### 圖片變形
- 原因：固定 height 但沒有 `object-fit`
- 檢查：使用 `height: auto` 或 `object-fit: cover`
